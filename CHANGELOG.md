# Changelog

## v1.0.1

### Fixed
- Исправлено восстановление соединения с MV210 после ошибок `Broken pipe`.
- Улучшено определение USB флешек для CSV export.
- Добавлена поддержка флешек без разделов (`/dev/sdX` вместо `/dev/sdX1`).
- Добавлены корректные права монтирования USB накопителей для пользователя сервиса `omega`.
- Улучшена стабильность работы Modbus TCP соединения.
- Добавлена принудительная переинициализация соединения MV210 после ошибок чтения.
- Улучшена совместимость с различными типами USB накопителей.

### Installation
- Исправлена инструкция копирования проекта из `starter_project/src`.
- Добавлено описание структуры проекта.
- Добавлено создание `/media/usb_export`.
- Добавлено назначение прав пользователю `omega`.

### System Preparation
- Добавлены рекомендации по отключению UAS для нестабильных USB-SATA bridge.
- Добавлены рекомендации по HDMI/headless режиму Raspberry Pi.
- Добавлены рекомендации по SQLite WAL mode.
- Добавлены рекомендации по проверке питания Raspberry Pi через `vcgencmd get_throttled`.

## v1.0.2 — Planned

### Planned
- Использование фиксированного пользователя `omega` вместо `os.getuid()/os.getgid()` для USB mount.
- Защита от записи export файлов при отсутствии mounted USB накопителя.
- Улучшенная диагностика USB export.
- Импорт конфигурации системы с USB флешки в режиме `WAIT`.
- Настройка параметров насоса через файл конфигурации на флешке:
  - размер плунжера;
  - ход плунжера;
  - число плунжеров;
  - передаточное число редуктора;
  - импульсы кардана на оборот;
  - коэффициент пересчёта оборотов двигателя.
- Настройки, импортированные с USB, сохраняются в SQLite `meta_state`.
- USB используется только как транспорт для передачи `config_import.txt`.
- Рабочие параметры системы читаются из БД, а не из файла на флешке.

- Запись диагностических параметров Raspberry Pi в системный журнал:
  - температура CPU;
  - `vcgencmd get_throttled`;
  - load average;
  - использование RAM;
  - свободное место на диске;
  - I/O wait;
  - ошибки USB/storage при наличии.
